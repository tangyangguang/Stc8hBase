# 新增模块准入标准

## 1. 目的

本库优先服务真实 STC8H1K08 小项目，不追求外设数量。新增基础能力或外设驱动前，必须先确认它确实属于可复用基础库，而不是某个业务项目的临时需求。

## 2. 必须满足

新增模块至少满足以下条件：

- 已有真实项目需求、已复现的基础库缺口，或多个项目明显会重复使用。
- 涉及芯片寄存器、引脚复用、电气特性、时序或协议时，已查官方或权威资料，并记录到 `docs/10_REFERENCES.md`。
- API 足够小，能说明“不做什么”。
- 不使用该模块时，不编译对应 `.c` 文件，不占用 ROM、RAM、外设、中断或板级初始化。
- 有最小示例或宿主机测试，能证明基本行为和资源占用。
- 资源报告记录编译文件、ROM/RAM、外设、中断和未使用符号检查结果。

## 3. 优先级

优先加入 MCU 生存性和项目高频基础能力，例如 WDT、外部中断、低功耗、必要的 Timer 辅助。

按真实项目再加入外设，例如 1-Wire/DS18B20、74HC595、24Cxx、RS485 方向控制。

默认不加入大型协议栈、图形库、菜单框架、设备树、自动注册、统一外设对象表或资源管理器。

## 4. 评审问题

新增模块前至少回答：

- 没有这个模块，当前真实项目会如何受阻？
- 这个能力是否可以由已有 GPIO/UART/I2C/SPI/PWM/ADC/Timer 组合完成？
- 是否会引入运行期分派、函数指针表、全局缓冲或隐藏初始化？
- 是否有官方或权威资料支撑寄存器和时序？
- 最小示例编译后，未使用模块是否仍能通过 map 符号检查？

如果答案不清楚，先写评估和测试计划，不直接写实现。
