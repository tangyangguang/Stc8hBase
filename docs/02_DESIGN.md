# 设计原则

## 1. 首要原则

高效率、低占用优先。

这是 STC8H 基础库，不是桌面软件式框架。每个模块都应该小、直接、可预期，并适合 8051 类 MCU 的资源约束。

本库坚持自己实现。外部成熟库只作为参考，用来完善建设要求和避免重复踩坑，不直接照搬整体架构。

## 2. 成功标准

一个模块满足以下条件时，才适合进入基础库：

- 解决多个项目都会重复遇到的问题。
- 边界清晰，不包含应用业务逻辑。
- 默认 RAM 占用低。
- 避免不必要的函数指针表、动态分配和深层抽象。
- 可以通过一个小示例编译和验证。
- 配置方式从头文件中就能看清楚。
- 不使用时不占用 ROM、RAM、外设或中断资源。
- 能按照既有模块边界扩展，不需要改动无关模块。

## 3. 设计取舍

### 3.1 优先使用编译期配置

如果某个选择在一个固件中是固定的，优先使用宏、常量和小型配置头文件。

只有同一个固件确实需要在运行时切换行为时，才使用运行期配置。

芯片型号、系统时钟、编译器差异和板级引脚应集中配置，避免散落在各个模块中。

### 3.2 存储占用显式化

驱动默认不应该分配较大的私有缓冲区。

如果模块需要状态或缓冲区，优先选择：

- 调用方提供对象。
- 调用方提供缓冲区。
- 硬件资源天然单例时，模块内部使用少量固定静态状态。

### 3.3 未使用模块零占用

基础库必须支持按需引入。

应用项目没有包含、编译或引用的模块，不应该产生任何资源占用。

设计时遵守：

- 模块之间避免隐式依赖。
- 不在公共头文件中定义会分配存储的对象。
- 不用全局自动注册机制。
- 不把多个无关驱动打包进同一个源文件。
- 示例和应用项目只加入实际使用的 `.c` 文件。

### 3.4 易扩展

基础库要方便扩展新的功能和外设，但扩展方式必须轻量。

新增模块优先通过新增独立 `.h/.c` 文件完成，不为了扩展能力引入大型框架。

扩展设计应遵守：

- 新模块放入对应层级：`core`、`hal`、`drivers` 或 `utils`。
- 新模块只依赖必要的下层模块。
- 不修改无关模块的公共 API。
- 不增加未使用模块的默认资源占用。
- 对常见外设优先提供最小可用能力，再根据真实项目需求补充。

### 3.5 编译期引脚绑定优先

8051 位操作对效率和代码体积很敏感。

高频 GPIO 路径优先使用板级宏或内联函数在编译期绑定具体引脚，让编译器尽量生成直接位操作。

允许保留 `port + pin` 形式的通用 API，但它只适合：

- 低频初始化。
- 调试工具。
- 示例中的非关键路径。

驱动实现中，按键扫描、EC11 扫描、TM1637 时序、软件 I2C、红外收发这类高频或时序敏感路径，不应依赖运行期端口分派。

这类驱动应优先设计成“状态机 + 板级引脚宏”模式：驱动维护状态和协议逻辑，板级代码负责具体 GPIO 读写。

### 3.6 HAL 保持薄

HAL 应该简化常见寄存器操作，但不应该隐藏芯片本身。

好的 HAL 风格：

- 小函数或宏。
- 直接对应 STC8H 硬件概念。
- 没有复杂对象模型。
- 没有统一外设大框架。

不好的 HAL 风格：

- 大型设备树。
- 多态驱动接口。
- 过度间接调用。
- 复杂初始化构造器。

### 3.7 优先支持轮询式使用

很多 STC8H 小项目都是简单主循环。驱动应该优先适合这种使用方式。

中断支持有价值，但默认不应强制依赖中断，除非该硬件功能天然需要中断。

### 3.8 避免隐藏开销

高频调用的 API 不应该偷偷做耗时操作。

例如：

- 按键扫描函数可以更新消抖状态。
- 显示写入函数可以等待 LCD 时序。
- getter 不应该刷新硬件，除非名字明确表达了这一点。

## 4. 兼容策略

本基础库不为了旧项目代码、旧 API、旧接线习惯或局部约定保留不合理兼容。

旧项目代码可以作为参考材料，但基础库应暴露当前更干净的 API。

本项目不考虑老项目 EEPROM/IAP 历史格式，不为旧数据结构增加迁移兼容。

但涉及 EEPROM、Flash、校准数据、保存参数、日志或统计时，示例和测试仍必须明确使用的地址范围，不能静默擦写未知区域。

## 5. 错误处理

错误处理要符合 MCU 资源约束。

正常运行中可能出现、且调用方可以处理的失败，使用简单状态值。

只由程序员误用造成的不可能状态，不额外增加复杂处理；除非检查成本很低，并且能明显帮助开发调试。

## 6. 命名

使用统一前缀，让应用代码能快速识别基础库符号。

建议前缀：

- `stc8h_`：core 和 HAL 模块。
- `drv_`：外设驱动模块。
- `util_`：通用工具模块。

头文件名与模块名保持一致。

示例：

```text
core/stc8h_types.h
hal/stc8h_gpio.h
drivers/drv_button.h
utils/util_ring_buffer.h
```
