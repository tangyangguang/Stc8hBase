# 模块规划

## 1. 分层

```text
应用项目
  drivers/
  utils/
  hal/
  core/
STC8H 硬件
```

上层可以依赖下层，下层不能依赖上层。

## 2. Core

Core 模块定义编译器兼容、通用类型和极小的芯片级辅助能力。

第一版实现模块：

| 模块 | 作用 |
| --- | --- |
| `stc8h_compiler` | 编译器关键字和兼容宏 |
| `stc8h_types` | 固定宽度类型和通用状态值 |
| `stc8h_config` | 项目级编译期配置 |
| `stc8h_delay` | 粗略忙等延时和 Timer0 1T 微秒级阻塞延时 |
| `stc8h_interrupt` | 小型中断开关辅助 |

Core 必须保持很小，不能演变成框架。

## 3. HAL

HAL 模块封装常见硬件操作，同时保留 STC8H 的硬件概念。

第一版实现模块：

| 模块 | 作用 |
| --- | --- |
| `stc8h_gpio` | 引脚模式、读、写、翻转 |
| `stc8h_timer` | 定时器配置和 tick 来源 |
| `stc8h_uart` | UART 初始化、发送、接收辅助 |
| `stc8h_i2c` | 软件 I2C 单总线 |
| `stc8h_spi` | SPI 基础收发 |
| `stc8h_pwm` | PWM 输出配置 |
| `stc8h_adc` | ADC 采样辅助 |
| `stc8h_eeprom` | 持久化参数存储支持 |
| `stc8h_wdt` | 看门狗启用、喂狗和复位标志辅助 |
| `stc8h_exti` | INT0-INT4 外部中断基础配置 |
| `stc8h_power` | idle 和 power-down 低功耗入口 |

HAL API 避免大型结构体，除非它能明显减少重复代码。

WDT、外部中断和低功耗模块只封装已核实的 MCU 基础能力，不做任务健康监控、电源管理框架或中断注册表。ISR 仍由板级或应用代码显式绑定。

第一版 I2C 明确采用软件 I2C 单总线，SDA/SCL 通过板级宏绑定。

SPI 第一版采用硬件 SPI 主机轮询实现，只定义基础收发能力，不引入软件 SPI 双实现。默认使用 P1.3/P1.4/P1.5 引脚组，硬件 SS 使用 `SSIG=1` 忽略，片选由板级或应用代码自行控制。

## 4. Drivers

Drivers 支持外部器件和常见板级外设。

第一版实现模块：

| 模块 | 作用 |
| --- | --- |
| `drv_button` | 按键消抖、按下/释放/长按事件 |
| `drv_ec11` | EC11 编码器方向识别、快速旋转加速和按键组合支持 |
| `drv_lcd1602` | I2C LCD1602 字符屏驱动 |
| `drv_led` | LED 有效电平和状态辅助 |
| `drv_buzzer` | 有源蜂鸣器有效电平辅助 |
| `drv_relay` | 继电器有效电平辅助 |
| `drv_tm1637` | 常见四位数码管模块 |
| `drv_ir_tx` | NEC 红外遥控发射协议编码 |
| `drv_ir_rx` | NEC 红外遥控接收协议解码 |

驱动应该暴露硬件无关的行为，具体引脚映射保持显式。

LCD1602 第一版面向 I2C 转接板，采用 `VCC`、`GND`、`SDA`、`SCL` 四线连接。

红外遥控第一版只支持常见 NEC 协议。`drv_ir_tx` 只输出普通帧和 repeat 帧的 `mark/space` 时序，`drv_ir_rx` 只消费板级捕获层测得的脉宽；真实硬件发射的 38kHz 载波、红外 LED 驱动、真实硬件接收的 GPIO/中断/Timer 由板级示例单独绑定并记录资源占用。

LED、蜂鸣器、继电器驱动第一版不直接操作 GPIO，只提供有效电平和状态辅助。实际 GPIO 输出由板级宏或示例代码完成。

新增外设驱动时，优先新增独立模块，不修改已有驱动。应用项目只编译实际用到的驱动文件，未使用的外设驱动不能带来资源占用。

## 5. Utils

工具模块必须小、依赖少。

第一版实现模块：

| 模块 | 作用 |
| --- | --- |
| `util_ring_buffer` | UART 和数据流缓冲 |
| `util_soft_timer` | 非阻塞时间间隔判断 |
| `util_crc` | CRC/校验辅助 |
| `util_filter` | 限幅和移位式 IIR 平滑 |

工具模块不能变成隐藏调度器或操作系统。

## 6. 示例要求

每个稳定模块应该有一个最小示例。

示例应该展示：

- 必要配置。
- 初始化。
- 主循环用法。
- 预期行为。

示例避免混入无关功能，方便直接复制。
