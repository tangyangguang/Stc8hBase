# 已确认决策

## 1. 项目定位

本项目是 STC8H 单片机项目的可复用基础库。

目标不是做大框架，而是沉淀多个小项目都会用到的基础能力和外设驱动。

## 2. 设计原则

已确认原则：

- 高效率优先。
- 低 ROM/RAM 占用优先。
- 未使用模块零占用。
- 方便扩展新的基础能力和外设。
- 不为了旧项目保留不合理兼容。
- 不为了兼容更多芯片增加主芯片复杂度。

## 3. 第一版范围

第一版不做“以后再实现”的功能规划，但采用里程碑交付。

当前已设计的基础能力和外设支持，都纳入第一版实现范围。

以后只有遇到真实新外设或新功能需求时，才新增模块。

第一版范围是基础库能力集合，不是单个应用项目的默认启用清单。每个应用项目只选用其中一部分模块。

里程碑 1 完成并通过资源报告后，即可作为可用基础库发布；后续模块按硬件验证成熟度继续进入第一版能力集合。

## 4. 主芯片

第一版正式主支持：

```text
STC8H1K08
```

第一版可评估兼容：

```text
STC8H1K17
```

其他芯片不预先设计支持。

## 5. 封装习惯

用户常用 `TSSOP20` 封装。

第一版示例和板级配置优先考虑 `TSSOP20` 封装下的引脚资源约束。

## 6. 系统时钟

第一版默认系统时钟：

```text
11.0592MHz
```

原因：

- 对传统 8051 串口波特率更友好。
- 适合老 Keil 项目和串口调试。
- 适合 UART 示例作为默认配置。

保留可选配置：

```text
12MHz
```

12MHz 适合定时和延时心算更直观的项目。

## 7. 开发工具链

新项目主工作流：

```text
macOS + SDCC + Makefile/PlatformIO + stcgal
```

老项目：

```text
Keil C51 继续可用
```

基础库源码必须同时兼容 SDCC 和 Keil C51。

兼容方式使用编译期宏，不引入运行期开销。

## 8. 使用方式

开发期：

```text
公共库引用
```

正式发布：

```text
复制稳定版本到项目中冻结
```

每个应用项目只编译实际使用的 `.c` 文件。

## 9. 不做的事情

第一版不做：

- OLED 驱动。
- 红外多协议通用框架。
- 红外学习型遥控码库。
- 运行期芯片识别。
- 设备树。
- 自动注册机制。
- 统一外设对象表。
- 隐藏调度器或操作系统。
- 为未来未知需求预留复杂扩展层。

## 10. 外设和测试补充

已确认：

- LCD1602 使用 I2C 转接板，4 线连接：`VCC`、`GND`、`SDA`、`SCL`。
- 红外接收头使用 `VS1838B` 或 `HS0038`。
- 红外协议第一版只支持 NEC。
- 当前没有逻辑分析仪，使用示波器做关键波形和时序验证。
- 不考虑任何老项目 EEPROM/IAP 数据格式，不为历史项目增加迁移兼容。
- I2C LCD1602 默认地址按 `0x27`，保留 `0x3F` 作为常见备用地址。
- UART 默认波特率按 `115200`，保留 `9600` 作为测试波特率。
- 红外发射管默认按三极管或 MOS 管外部驱动设计。
- 蜂鸣器默认按有源蜂鸣器处理，无源蜂鸣器通过 PWM 模块驱动。
- I2C 第一版优先软件 I2C，以适配 `TSSOP20` 引脚灵活性。
