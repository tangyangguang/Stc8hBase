# 项目规则

- 本仓库是 STC8H 单片机项目的可复用基础库。
- 优先考虑运行效率和 ROM/RAM 低占用，而不是通用抽象。
- 添加实现代码前，先讨论并记录模块设计。
- 编码和调试芯片、寄存器、外设前，必须先查官方资料；找不到官方资料时，使用权威资料并记录来源。
- 维护参考资料列表；高频使用资料可存入本地，但必须控制数量和体积。
- API 保持小、明确、稳定。
- 不为不合理的旧设计增加兼容层。
- 涉及持久化数据、校准参数或配置结构变化时，必须先给出迁移、备份或恢复方案。
- 每次完成新功能、优化或完善后，提交并推送相关改动。
- 每次回复用户时，说明当前剩余工作。
- 本文件保持在 100 行以内。
